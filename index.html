Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

In order to consult a potential client on how to protect his/her website from attacks, I should firstly state the importance of website security. It can be simplified into a website without security like a house without locked doors that any malicious attackers can easily get into. This consequently causes the loss of private and confidential information such as email addresses, passwords and credit card details. The website can be out of control and even the business may be in danger.

I should also give a quick sum of some common threats such as Cross-site Scripting (XSS), SQL injection, Cross-Site Request Forgery (CSRF), etc. for the clients to get an overall view before coming up with possible protections to keep the website safe and sound. They can be:

  • Sanitizing all data before being passed to next steps. Never trust any input data which is user-originated. Be careful with files uploaded.
  • Encouraging to enable HTTPS and HTTP Strict Transport Security (HSTS) protocols. This will ensure data sent between the client and server to be protected from attackers.
  • Taking the advantages of available vulnerability scanner tools to test the security of the website. A bug bounty program can be a good way to fix bugs.
  • Strengthening passwords and using two-factor authentication to log in.
  • Using a server-side web framework. Being up-to-date with the most popular threats.
  • Eliminating storing sensitive information like credit card details.
  • Always being ready for the worst with regular backups.
